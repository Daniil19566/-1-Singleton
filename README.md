2. Реализовать класс Singleton:
• Класс
должен иметь приватный конструктор и статическое свойство (или метод), возвращающее единственный экземпляр.
•
Необходимо обеспечить потокобезопасность (например, через двойную проверку if (_instance == null) и lock).
•
Предусмотреть какую-то бизнес-логику внутри Singleton (например, подсчёт обращений к экземпляру или хранение
конфигурации приложения).
3. Продемонстрировать реализацию:
• Можно создать несколько потоков, каждый из которых «обращается» к Singleton и выводит в консоль текущее количество обращений.
